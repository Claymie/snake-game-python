# Трекер места в конкурсных списках вузов

Следит за местом одного или нескольких абитуриентов (по коду, который
присваивают на Госуслугах) в открытых конкурсных списках вузов и
присылает уведомление в Telegram, когда место меняется — каждому лично
в его чат. Пароль/логин от Госуслуг нигде не используется — конкурсные
списки публикуются вузами в открытом доступе специально для того, чтобы
абитуриенты сами проверяли своё место, без входа в личный кабинет.

## Как это работает

1. GitHub Actions по расписанию (по умолчанию — раз в 2 часа, файл
   `.github/workflows/track_admission.yml`) запускает `main.py`.
2. Скрипт открывает страницу конкурсного списка каждого направления из
   `config.yaml` в headless-браузере (Playwright/Chromium — нужен, потому
   что часть сайтов рисует таблицы через JavaScript) один раз на
   направление, и в уже загруженной странице ищет код **каждого**
   зарегистрированного пользователя (`users` в `config.yaml`) — открывать
   страницу повторно на каждого человека не нужно.
3. Текущее место/общее число участников каждого пользователя сохраняется
   в `university_tracker/data/state.json` (файл коммитится обратно в
   репозиторий, так и хранится история).
4. Если у кого-то место изменилось, он появился/пропал из списка —
   именно ему прилетает сообщение в Telegram (остальным, если у них ничего
   не изменилось, — тишина).
5. Отдельно, каждые 5 минут (`.github/workflows/telegram_on_demand.yml`),
   GitHub Actions дёшево проверяет через Telegram API, не написал ли
   кто-то из зарегистрированных пользователей боту `/start` (или
   `/check`, `/status`, `проверить`) — и если да, полноценно запускает
   проверку и присылает отчёт именно этому человеку в течение примерно
   минуты, не дожидаясь планового прогона раз в 2 часа.

## ⚠️ Важное ограничение

Код писался в изолированной песочнице, у которой сетевая политика
блокирует прямой доступ к сайтам вузов — то есть я не могу сам открыть
реальные страницы со списками. По реальным прогонам (через GitHub
Actions) выяснилось:

- **sibsutis.ru блокирует запросы с серверов GitHub Actions** (сервер
  сбрасывает соединение, `net::ERR_EMPTY_RESPONSE`) — не отслеживается.
- **nstu.ru открывается нормально.** Сейчас отслеживаются 5 направлений
  факультета АВТФ, каждое по своей прямой ссылке (`entrance_list?comp_name=АВТФ.N`).
- Список на странице nstu.ru — не HTML-таблица, а карточки с текстом
  вида `ID профиля ЕПГУ: 1339447`; порядок карточек в документе
  соответствует месту в конкурсном списке. За разбор такого формата
  отвечает `find_applicant_by_id_epgu()` в `parser.py`
  (`mode: "id_epgu"` в конфиге), покрыт тестами в
  `tests/test_parser.py`.
- Если список подгружается по прокрутке (бесконечный скролл), в
  `fetcher.py` есть автопрокрутка (`autoscroll: true`) — если весь
  список и так есть в HTML сразу, она просто ничего не меняет.

Точная структура страницы проверялась только по присланному скриншоту,
не вживую — если после реального прогона направление не найдётся,
пришли лог (Actions → workflow run → шаг «Проверить конкурсные
списки», строка `Текущий статус по каждому направлению`), и поправлю.

## Настройка

### 1. Создать Telegram-бота

1. Напиши [@BotFather](https://t.me/BotFather) в Telegram, команда
   `/newbot`, следуй инструкциям — получишь `TELEGRAM_BOT_TOKEN`.
2. Напиши своему новому боту любое сообщение (`/start`).
3. Узнай свой `chat_id`: открой в браузере
   `https://api.telegram.org/bot<TELEGRAM_BOT_TOKEN>/getUpdates`
   и найди `"chat":{"id": ...}` в ответе.

### 2. Добавить секрет в репозиторий

В настройках репозитория: **Settings → Secrets and variables → Actions →
New repository secret**. Добавить:

- `TELEGRAM_BOT_TOKEN` — токен бота из шага 1

Код абитуриента и chat_id теперь не в секретах, а в `config.yaml` (раздел
`users`, см. ниже) — это не пароли, а просто данные для поиска в открытых
списках, и хранить список из нескольких человек в YAML удобнее, чем в
плоских секретах.

### 3. Пользователи и направления (уже настроено)

`config.yaml` содержит:

- `users` — список людей, за которых нужно следить: `name` (любое имя для
  логов), `applicant_code` (код с Госуслуг) и `chat_id` (Telegram chat_id
  этого человека).
- `programs` — список направлений с прямыми ссылками на списки: 5 у НГТУ
  (АВТФ) и 3 у НГУ. У НГУ ссылка вида
  `abiturient.nsu.ru/bachelor?faculty=..&direction=..&condition=..&type=0` —
  получена вручную (выбрать фильтры на сайте → скопировать ссылку из
  адресной строки), так же как и ссылки НГТУ.

Поля записи в `programs`:

- `url` — прямая ссылка на список конкретного направления.
- `mode: "id_epgu"` — использовать парсер под карточки НГТУ
  (`ID профиля ЕПГУ: ...`). Для сайтов с обычными HTML-таблицами это поле
  не нужно (по умолчанию используется общий поиск по таблицам/тексту —
  им сейчас пользуется НГУ, формат его страницы не проверялся вживую).
- `click_sequence` — список текстов, по которым скрипт кликает по очереди
  перед чтением страницы (открыть выпадающий список → выбрать значение →
  ... → нажать кнопку отправки формы). Нужен только если у сайта нет
  прямых ссылок на конкретный список (сейчас не используется — и у НГТУ,
  и у НГУ нашлись прямые ссылки).
- `autoscroll: true` — прокручивать страницу до конца перед чтением (на
  случай подгрузки по скроллу).

Если понадобится добавить ещё вуз/направление — пришли мне прямую ссылку
на список (лучше всего — как для НГУ и НГТУ: выбрать фильтры на сайте
руками и скопировать итоговый URL); если формат страницы окажется другим,
поправлю парсер под него.

### 4. Как добавить друга

1. Пусть он напишет что угодно боту @egesoon_bot (например `/start`).
2. Узнай его `chat_id` тем же способом, что и свой:
   `https://api.telegram.org/bot<TELEGRAM_BOT_TOKEN>/getUpdates`, найти
   `"chat":{"id": ...}` в его сообщении.
3. Узнай его код абитуриента с Госуслуг.
4. Добавь запись в `users` в `config.yaml`:
   ```yaml
   - name: "Имя друга"
     applicant_code: "..."
     chat_id: "..."
   ```
5. Запушь/попроси смержить изменение — дальше друг получает уведомления и
   может сам писать `/start` боту, не видя твоих данных (каждому в его
   личный чат приходит только его собственный статус).

### 5. Проверить вручную

Actions → «Отслеживание места в конкурсных списках вузов» →
**Run workflow**. После первого прогона глянь `state.json` и лог — если
для каких-то направлений место не нашлось, пришли мне детали.

### 6. Проверка по запросу из Telegram

Любому зарегистрированному в `users` человеку достаточно написать боту
`/start` (или `/check`, `/status`, `проверить`, либо нажать кнопку
"Проверить статус" внизу чата — она появляется сама после первого ответа
бота).

Команды `/start`, `/check`, `/status` также регистрируются в меню бота
(значок "/" в поле ввода Telegram) автоматически, через
`setMyCommands` — руками через BotFather ничего настраивать не нужно.

**Мгновенный ответ (рекомендуется):** настрой Cloudflare Worker-вебхук
по инструкции в [`university_tracker/webhook/README.md`](webhook/README.md)
— тогда ответ приходит за 1-2 минуты сразу после сообщения, без ожидания
расписания GitHub.

⚠️ Без вебхука: на этом аккаунте GitHub `schedule`-триггеры срабатывают
сами очень редко и непредсказуемо (антифрод-ограничение GitHub на
автоматические расписания для новых аккаунтов, а не баг в коде) —
`telegram_on_demand.yml` без вебхука будет отвечать с задержкой от минут
до часов. Пока вебхук не настроен, можно запускать вручную: Actions →
«Ответ боту в Telegram по запросу (запасной опрос)» → **Run workflow**.

## Локальный запуск (без GitHub Actions)

```bash
cd university_tracker
pip install -r requirements.txt
python -m playwright install chromium
TELEGRAM_BOT_TOKEN=xxx \
python main.py --force-notify   # --force-notify шлёт полный статус всем users, даже без изменений
```
