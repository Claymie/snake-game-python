# Трекер места в конкурсных списках вузов

Следит за твоим местом в открытых конкурсных списках вузов (по коду
абитуриента, который присваивают на Госуслугах) и присылает уведомление
в Telegram, когда место меняется. Пароль/логин от Госуслуг нигде не
используется — конкурсные списки публикуются вузами в открытом доступе
специально для того, чтобы абитуриенты сами проверяли своё место, без
входа в личный кабинет.

## Как это работает

1. GitHub Actions по расписанию (по умолчанию — раз в 2 часа, файл
   `.github/workflows/track_admission.yml`) запускает `main.py`.
2. Скрипт открывает страницу конкурсного списка каждого направления из
   `config.yaml` в headless-браузере (Playwright/Chromium — нужен, потому
   что часть сайтов рисует таблицы через JavaScript) и ищет строку с твоим
   кодом.
3. Текущее место/общее число участников сохраняется в
   `university_tracker/data/state.json` (файл коммитится обратно в
   репозиторий, так и хранится история).
4. Если место изменилось, ты появился/пропал из списка — прилетает
   сообщение в Telegram.
5. Отдельно, каждые 5 минут (`.github/workflows/telegram_on_demand.yml`),
   GitHub Actions дёшево проверяет через Telegram API, не написал ли ты
   боту `/start` (или `/check`, `/status`, `проверить`) — и если да,
   полноценно запускает проверку и присылает отчёт в течение примерно
   минуты, не дожидаясь ближайшего планового прогона раз в 2 часа.

## ⚠️ Важное ограничение

Код писался в изолированной песочнице, у которой сетевая политика
блокирует прямой доступ к сайтам вузов — то есть я не могу сам открыть
реальные страницы со списками. По реальным прогонам (через GitHub
Actions) выяснилось:

- **sibsutis.ru блокирует запросы с серверов GitHub Actions** (сервер
  сбрасывает соединение, `net::ERR_EMPTY_RESPONSE`) — не отслеживается.
- **nstu.ru открывается нормально.** Сейчас отслеживаются 5 направлений
  факультета АВТФ, каждое по своей прямой ссылке (`entrance_list?comp_name=АВТФ.N`).
- Список на странице nstu.ru — не HTML-таблица, а карточки с текстом
  вида `ID профиля ЕПГУ: 1339447`; порядок карточек в документе
  соответствует месту в конкурсном списке. За разбор такого формата
  отвечает `find_applicant_by_id_epgu()` в `parser.py`
  (`mode: "id_epgu"` в конфиге), покрыт тестами в
  `tests/test_parser.py`.
- Если список подгружается по прокрутке (бесконечный скролл), в
  `fetcher.py` есть автопрокрутка (`autoscroll: true`) — если весь
  список и так есть в HTML сразу, она просто ничего не меняет.

Точная структура страницы проверялась только по присланному скриншоту,
не вживую — если после реального прогона направление не найдётся,
пришли лог (Actions → workflow run → шаг «Проверить конкурсные
списки», строка `Текущий статус по каждому направлению`), и поправлю.

## Настройка

### 1. Создать Telegram-бота

1. Напиши [@BotFather](https://t.me/BotFather) в Telegram, команда
   `/newbot`, следуй инструкциям — получишь `TELEGRAM_BOT_TOKEN`.
2. Напиши своему новому боту любое сообщение (`/start`).
3. Узнай свой `chat_id`: открой в браузере
   `https://api.telegram.org/bot<TELEGRAM_BOT_TOKEN>/getUpdates`
   и найди `"chat":{"id": ...}` в ответе.

### 2. Добавить секреты в репозиторий

В настройках репозитория: **Settings → Secrets and variables → Actions →
New repository secret**. Добавить:

- `APPLICANT_CODE` — твой код абитуриента (`1339447`)
- `TELEGRAM_BOT_TOKEN` — токен бота из шага 1
- `TELEGRAM_CHAT_ID` — chat_id из шага 1

### 3. Настройка направлений (уже сделана для НГТУ/АВТФ)

`config.yaml` отслеживает 5 направлений факультета АВТФ в НГТУ, у каждого
своя прямая ссылка на список. Поля записи:

- `url` — прямая ссылка на список конкретного направления.
- `mode: "id_epgu"` — использовать парсер под карточки НГТУ
  (`ID профиля ЕПГУ: ...`). Для сайтов с обычными HTML-таблицами это поле
  не нужно (по умолчанию используется общий поиск по таблицам).
- `autoscroll: true` — прокручивать страницу до конца перед чтением (на
  случай подгрузки по скроллу).

Если понадобится добавить ещё вуз/направление — пришли мне прямую ссылку
на список; если формат страницы окажется другим (обычная таблица или
что-то ещё), поправлю парсер под него.

### 4. Проверить вручную

Actions → «Отслеживание места в конкурсных списках вузов» →
**Run workflow**. После первого прогона глянь `state.json` и лог — если
для каких-то направлений место не нашлось, пришли мне детали.

### 5. Проверка по запросу из Telegram

После того как секреты добавлены и workflow хотя бы раз смержен в `main`,
достаточно написать боту `/start` (или `/check`, `/status`, `проверить`) —
в течение ~5 минут (частота опроса в `telegram_on_demand.yml`) придёт
сообщение "Проверяю...", а через ~минуту — полный отчёт по всем
направлениям. Ничего заходить в GitHub не нужно.

## Локальный запуск (без GitHub Actions)

```bash
cd university_tracker
pip install -r requirements.txt
python -m playwright install chromium
APPLICANT_CODE=1339447 \
TELEGRAM_BOT_TOKEN=xxx \
TELEGRAM_CHAT_ID=xxx \
python main.py --force-notify   # --force-notify шлёт сводку даже без изменений
```
